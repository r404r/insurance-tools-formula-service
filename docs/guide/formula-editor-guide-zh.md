# 公式编辑器操作指南

本指南面向非技术人员，帮助您使用保险计算式引擎的可视化公式编辑器。无需编程经验即可创建、编辑和测试计算公式。

---

## 1. 界面概览

打开公式编辑器后，您会看到以下界面：

![可视化编辑器概览](images/02-visual-editor.png)

界面分为以下几个区域：

| 区域 | 位置 | 说明 |
|------|------|------|
| **顶栏** | 页面顶部 | 公式名称、分类选择、版本号、模式切换（Visual / Text）、保存按钮 |
| **节点调色板** | 左侧栏 | 可拖拽的节点类型列表 |
| **画布** | 中央 | 公式的可视化编辑区域，可拖拽节点并连线 |
| **属性面板** | 右侧栏 | 选中节点后显示其属性，可编辑配置 |
| **测试面板** | 页面底部 | 输入测试参数并执行计算 |

## 2. 节点类型一览

公式由多种类型的"节点"组成，每种节点代表一种计算元素：

| 图标 | 名称 | 用途 | 举例 |
|------|------|------|------|
| `x` | **变量** | 公式的输入参数 | 年龄、保额、费率 |
| `#` | **常量** | 固定的数值 | 1、100、3.14 |
| `+-` | **运算符** | 加减乘除等算术运算 | 保额 × 费率 |
| `f(x)` | **函数** | 数学函数 | round（四舍五入）、sqrt（平方根） |
| `{}` | **子公式** | 引用其他已有的公式 | 引用"基础费率计算"公式 |
| `[]` | **表查询** | 从数据表中查找数值 | 根据年龄查询死亡率 |
| `?` | **条件** | 根据条件选择不同的值 | 如果年龄≥65 则…否则… |
| `Σ` | **聚合** | 将多个值合并为一个 | 求和、平均 |
| `↺` | **循环** | 按范围重复计算 | 对 t=1 到 n 求和 |

## 3. 基本操作

### 3.1 创建节点

1. 在左侧的**节点调色板**中找到需要的节点类型
2. 用鼠标**按住并拖拽**到画布中
3. 松开鼠标，节点就创建好了

### 3.2 连接节点

节点之间通过**端口**连接，形成计算流程：

- 每个节点**右侧**有一个输出端口（Out）
- 每个节点**左侧**有一个或多个输入端口（如 Left、Right）
- 从一个节点的输出端口**拖动**到另一个节点的输入端口即可连线

**连线规则：**
- 数据从左到右流动（输入 → 处理 → 输出）
- 不能将节点连接到自身
- 每个输入端口只能接收一条连线

### 3.3 编辑节点属性

1. **单击**画布中的节点将其选中
2. 右侧的**属性面板**会显示该节点的配置项
3. 根据节点类型修改相应的属性

![节点属性面板](images/03-node-selected.png)

**各节点属性说明：**

| 节点类型 | 需要设置的属性 |
|---------|--------------|
| 变量 | 变量名称（如 `age`）、数据类型 |
| 常量 | 数值（如 `100`） |
| 运算符 | 运算类型：加(+)、减(-)、乘(×)、除(÷)、幂(^)、取模(%) |
| 函数 | 函数名称：round、floor、ceil、abs、min、max、sqrt、ln、exp |
| 子公式 | 选择要引用的公式、可指定版本 |
| 表查询 | 选择数据表、设置查找键列、输出列 |
| 条件 | 比较运算符：==、!=、>、>=、<、<= |
| 循环 | 循环体公式、迭代变量名、聚合方式 |

### 3.4 删除节点或连线

1. **单击**要删除的节点或连线将其选中
2. 按键盘上的 **Delete** 键删除

### 3.5 自动布局

如果节点位置混乱，点击画布上方的**"自动布局"**按钮，系统会自动整理节点排列。

## 4. 构建公式示例

### 示例：计算 BMI（身体质量指数）

BMI = 体重 ÷ 身高²

**步骤：**

1. 拖入一个**变量**节点，命名为 `weight`（体重）
2. 拖入一个**变量**节点，命名为 `height`（身高）
3. 拖入一个**常量**节点，值设为 `2`
4. 拖入一个**运算符**节点，选择"幂(^)"
   - 将 `height` 连到 Left 端口
   - 将常量 `2` 连到 Right 端口
5. 拖入一个**运算符**节点，选择"除(÷)"
   - 将 `weight` 连到 Left 端口
   - 将幂运算结果连到 Right 端口

完成后，画布上会形成一个从左到右的计算流程图。

## 5. 进阶功能

### 5.1 子公式引用

当公式变得复杂时，可以将一部分计算拆分成独立的子公式，然后在主公式中引用它。

1. 先创建并保存子公式
2. 在主公式中拖入一个**子公式**节点
3. 在属性面板中选择要引用的公式

### 5.2 表查询

可以从预定义的数据表中查找数值（如费率表、死亡率表）。

1. 拖入一个**表查询**节点
2. 在属性面板中选择数据表
3. 设置查找键（用于定位数据行的列）
4. 设置输出列（要取出的数值列）
5. 将查找键对应的值连接到输入端口

### 5.3 条件分支

用于根据不同条件执行不同的计算。

1. 拖入一个**条件**节点
2. 设置比较运算符（如 `>=`）
3. 连接四个输入端口：
   - **If**：被比较的值（如 年龄）
   - **Cmp**：比较基准（如 常量 65）
   - **Then**：条件成立时的值
   - **Else**：条件不成立时的值

### 5.4 循环（Loop）

用于按年度、期间等进行重复计算，是保险数理计算的核心功能。

![循环节点示例](images/05-annuity-visual.png)

**基本循环（求和/求积）：**

1. 先创建循环体（body）子公式并保存发布
2. 在主公式中拖入一个**循环**节点
3. 设置属性：
   - **循环体公式**：选择刚创建的子公式
   - **迭代变量**：循环中的计数器名称（如 `t`）
   - **聚合方式**：sum（求和）、product（求积）等
4. 连接 Start 和 End 端口（循环范围）

**递推循环（Fold）：**

用于每一步依赖上一步结果的计算（如准备金递推）。

![Fold 循环示例](images/06-reserve-fold.png)

1. 拖入循环节点，聚合方式选择 **Fold**
2. 设置**累积变量名**（如 `V`）和**初始值**（如 `0`）
3. 循环体公式中使用 `V`（上一步的结果）和 `t`（当前步骤）

## 6. 文本编辑模式

除了可视化编辑，还可以通过文本方式编写公式。

![文本编辑器](images/04-text-editor.png)

### 6.1 切换到文本模式

点击页面顶部的**"Text Editor"**标签即可切换。

### 6.2 文本语法

| 写法 | 含义 |
|------|------|
| `a + b` | 加法 |
| `a * b` | 乘法 |
| `a / b` | 除法 |
| `a ^ 2` | 幂运算 |
| `round(x, 2)` | 四舍五入到小数点后 2 位 |
| `sqrt(x)` | 平方根 |
| `if a >= b then x else y` | 条件判断 |
| `lookup("表名", key)` | 表查询 |
| `subFormula("公式ID")` | 引用子公式 |
| `sum_loop("公式ID", t, 1, n)` | 求和循环 |
| `product_loop("公式ID", t, 1, n)` | 求积循环 |
| `fold_loop("公式ID", t, 0, n, V, 0)` | 递推循环 |

### 6.3 LaTeX 预览

在文本编辑模式下，编辑区下方会自动显示公式的**数学排版预览**（LaTeX 渲染）。

例如输入 `sum_loop("body", t, 1, n)` 会显示为：

$$\sum_{t=1}^{n} \text{body}(t)$$

### 6.4 LaTeX 输入模式

点击编辑区上方的 **"LaTeX"** 标签，可以直接输入 LaTeX 数学表达式，系统会自动将其转换为公式文本。

### 6.5 应用到图形

在文本模式中修改公式后，点击右上角的**"Apply to Graph"**按钮，即可将文本转换回可视化图形。

### 6.6 文本模式的限制

部分节点类型还不能在文本编辑器中往返编辑。如果你的公式包含以下节点，建议**只用可视化编辑器**：

- **Loop 节点**（`sum_loop` / `product_loop` / `fold_loop` 等）：部分 loop 配置项（`inclusiveEnd`、`maxIterations`、`version`）只在可视化模式下可编辑，文本模式无法表达。触发限制时编辑器会显示提示。
- **复合 Conditional 节点**（包含多个条件用 `AND` / `OR` / `NOT` 组合，task #039 引入）：文本语法目前没有 `and` / `or` / `not` 关键字，所以使用了复合条件的公式只能用可视化编辑器编辑——切换到文本模式时会显示明确的错误提示，引导你切回。

引擎层面的全部已知限制（包括统计分布函数、日期算术、表的跨行聚合、单次调用无状态等）记录在项目 [`README.md` § Known Limitations](../../README.md#known-limitations) 章节。

## 8. 保存与版本管理

### 保存公式

编辑完成后，点击右上角的**"Save"**按钮保存。系统会先执行验证：
- 如果有错误（如缺少连线），会显示红色提示
- 如果验证通过，显示绿色的"Saved"

### 版本历史

每次保存会创建一个新版本。点击顶栏的**"Version History"**可查看所有历史版本。

## 常见问题

**Q: 为什么保存时提示错误？**
检查是否所有节点的必填端口都已连线。常见问题：运算符节点缺少 Left 或 Right 输入。

**Q: 循环节点为什么报错？**
确认：(1) 已选择循环体公式 (2) 迭代变量名已填写 (3) Start 和 End 端口已连线。

**Q: 文本模式的公式显示不正确？**
文本模式会把所有没有连线的变量节点当作独立输出显示，这是正常现象。实际计算时这些变量会从输入参数中获取值。
