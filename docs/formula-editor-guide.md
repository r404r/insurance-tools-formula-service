# 公式编辑器使用手册

## 1. 概述

公式编辑器是保险公式计算引擎的核心功能，用于创建、编辑和测试保险计算公式。编辑器支持**可视化拖拽**和**文本表达式**两种编辑模式，公式以 DAG（有向无环图）格式存储，可自动并行计算。

---

## 2. 公式管理

### 2.1 公式列表

登录后进入公式列表页面，可执行以下操作：

| 操作 | 说明 | 权限 |
|------|------|------|
| 搜索公式 | 顶部搜索框，按名称关键字过滤 | 所有角色 |
| 领域过滤 | 切换标签页：全部 / 人寿保险 / 财产保险 / 车险 | 所有角色 |
| 新建公式 | 点击右上角「新建公式」按钮 | Admin, Editor |
| 进入编辑 | 点击公式列表中的任意一行 | 所有角色（只读/可编辑取决于角色） |

### 2.2 新建公式

点击「新建公式」后弹出创建对话框：

| 字段 | 必填 | 说明 |
|------|------|------|
| 公式名称 | 是 | 公式的唯一标识名称 |
| 保险领域 | 是 | 人寿保险(life) / 财产保险(property) / 车险(auto) |
| 描述 | 否 | 公式用途说明 |

创建成功后自动跳转到编辑器页面。

## 4. 可视化编辑模式

### 4.1 节点类型

编辑器左侧「节点类型」面板提供 8 种可拖拽的节点：

| 图标 | 节点类型 | 中文名 | 用途 | 节点颜色 |
|------|---------|--------|------|---------|
| `x` | variable | 变量 | 定义输入参数（如 age、sumAssured） | 蓝色 |
| `#` | constant | 常量 | 固定数值（如 1、0.035、100） | 琥珀色 |
| `+-` | operator | 运算符 | 四则运算和幂运算 | 粉色 |
| `f(x)` | function | 函数 | 数学函数（round、sqrt 等） | 绿色 |
| `{}` | subFormula | 子公式 | 引用其他公式的计算结果 | 靛蓝色 |
| `[]` | tableLookup | 查表 | 从查找表（如生命表）中取值 | 紫色 |
| `?` | conditional | 条件 | 条件判断分支 | 橙色 |
| `E` | aggregate | 聚合 | 聚合运算（求和、平均等） | 青绿色 |

### 4.2 基本操作

#### 添加节点
1. 从左侧面板**拖拽**节点类型到画布区域
2. 节点会在鼠标释放位置创建
3. 节点自动显示类型对应的颜色和标签

#### 连接节点
1. 将鼠标移到节点底部的**输出端口**（小圆点）
2. **按住拖拽**到目标节点顶部的**输入端口**
3. 释放后自动创建带箭头的连线

#### 选中和编辑节点
1. **单击**画布中的节点进行选中（出现蓝色光晕）
2. 选中后右侧属性面板显示该节点的配置项
3. 修改属性后节点标签自动更新
4. **单击空白区域**取消选中

#### 移动节点
- 直接**拖拽**画布中的节点到新位置

#### 自动布局
- 点击画布右上角「Auto Layout」按钮
- 使用 dagre 算法自动排列所有节点，避免重叠

#### 画布控制
- **滚轮**：缩放
- **拖拽空白区域**：平移
- 左下角控制栏：放大(+)、缩小(-)、适应窗口、锁定
- 右下角：小地图总览

### 4.3 节点配置详解

选中节点后，右侧属性面板显示对应的配置项：

#### Variable（变量）

| 属性 | 说明 | 示例 |
|------|------|------|
| Name | 变量名称，对应计算时传入的参数名 | `sumAssured`、`age`、`qx` |
| Data Type | 数据类型 | Integer / Decimal / String / Boolean |

> 变量节点在画布上显示为变量名（如 `sumAssured`）

#### Constant（常量）

| 属性 | 说明 | 示例 |
|------|------|------|
| Value | 常量值（字符串格式，支持高精度） | `1`、`0.035`、`1000000` |

> 常量节点在画布上直接显示值（如 `1`、`0.035`）

#### Operator（运算符）

| 属性 | 选项 | 画布显示 |
|------|------|---------|
| Operator | + Add (加) | `+` |
| | - Subtract (减) | `−` |
| | * Multiply (乘) | `×` |
| | / Divide (除) | `÷` |
| | ^ Power (幂) | `^` |
| | % Modulo (取模) | `%` |

> 运算符节点接受**两个输入**（left、right），产出一个输出

#### Function（函数）

| 属性 | 可选函数 | 说明 |
|------|---------|------|
| Function | `round` | 四舍五入（可指定小数位数） |
| | `floor` | 向下取整 |
| | `ceil` | 向上取整 |
| | `abs` | 绝对值 |
| | `min` | 最小值 |
| | `max` | 最大值 |
| | `sqrt` | 平方根 |
| | `ln` | 自然对数 |
| | `exp` | 指数函数 |

> 函数节点在画布上显示为函数名，如 `round(18)` 表示保留18位小数

#### TableLookup（查表）

| 属性 | 说明 | 示例 |
|------|------|------|
| Table ID | 查找表标识 | `mortality-2024` |
| Lookup Key | 查找键 | `age` |
| Column | 结果列名 | `qx` |

> 用于从预加载的查找表（如生命表、费率表）中查询值

#### Conditional（条件）

| 属性 | 选项 | 含义 |
|------|------|------|
| Comparator | `== Equal` | 等于 |
| | `!= Not Equal` | 不等于 |
| | `> Greater` | 大于 |
| | `>= Greater/Equal` | 大于等于 |
| | `< Less` | 小于 |
| | `<= Less/Equal` | 小于等于 |

> 条件节点接受 condition、thenValue、elseValue 三个输入

#### Aggregate（聚合）

| 属性 | 选项 | 说明 |
|------|------|------|
| Aggregation | Sum | 求和 |
| | Average | 平均值 |
| | Count | 计数 |
| | Product | 连乘 |

#### SubFormula（子公式）

- 通过输入其他公式的 ID 来引用外部公式的计算结果
- 可指定引用特定版本

## 5. 文本编辑模式

点击顶部「文本编辑」切换到文本模式。

### 5.1 文本表达式语法

支持标准数学表达式风格：

```
round(lookup(mortalityTable, age) * sumAssured, 18)
```

### 5.2 支持的运算和函数

- 四则运算：`+` `-` `*` `/`
- 幂运算：`^`
- 内置函数：`round()` `floor()` `ceil()` `abs()` `min()` `max()` `sqrt()` `ln()` `exp()`
- 查表函数：`lookup(tableName, key)`
- 条件表达式：`if/then/else`

### 5.3 操作

| 按钮 | 说明 |
|------|------|
| 验证 | 检查文本表达式语法是否正确 |

> 注意：文本模式与可视化模式通过后端 Pratt 解析器实现双向转换

## 6. 保存与版本管理

### 6.1 保存公式

1. 在编辑器中完成节点编辑和连线
2. 点击右上角「保存」按钮
3. 系统自动创建新版本（Draft 状态），版本号自动递增
4. 保存成功后显示「已保存」提示，版本号更新

### 6.2 版本状态

每个版本有三种状态，构成状态机：

```
    创建
     ↓
   Draft (草稿) ──→ Published (已发布) ──→ Archived (已归档)
     │                                        ↑
     └────────────────────────────────────────┘
```

| 状态 | 颜色标识 | 说明 |
|------|---------|------|
| Draft (草稿) | 黄色 | 新保存的版本，可继续编辑 |
| Published (已发布) | 绿色 | 正式生效版本，每个公式同时只有一个 |
| Archived (已归档) | 灰色 | 历史版本，只读 |

### 6.3 版本管理操作

点击编辑器中的「版本历史」链接进入版本管理页面：

| 操作 | 条件 | 说明 |
|------|------|------|
| 发布 | 当前为 Draft | 将版本设为 Published（自动归档之前的 Published 版本） |
| 归档 | 当前为 Draft 或 Published | 将版本设为 Archived |

## 7. 测试公式

编辑器底部提供测试面板：

### 7.1 操作步骤

1. 在输入框中填入 JSON 格式的参数，例如：
   ```json
   {"sumAssured": "1000000", "qx": "0.001", "interestRate": "0.035"}
   ```
2. 点击「计算」按钮
3. 结果以 JSON 格式显示在右侧

### 7.2 注意事项

- 所有数值以**字符串**格式传入，避免精度丢失
- 参数名必须与公式中 Variable 节点的 `name` 属性一致
- 测试使用当前公式的最新 Published 版本计算

## 8. 示例：构建寿险净保费公式

目标公式：`premium = sumAssured × qx ÷ (1 + interestRate)`

### 步骤 1：添加输入变量

从左侧拖入 3 个 **Variable** 节点，分别配置：
- `sumAssured`（保额，Decimal）
- `qx`（死亡率，Decimal）
- `interestRate`（预定利率，Decimal）

### 步骤 2：添加常量

拖入 1 个 **Constant** 节点，设置 Value = `1`

### 步骤 3：添加运算节点

拖入 3 个 **Operator** 节点：
1. `+` — 计算 `1 + interestRate`
2. `÷` — 计算 `1 ÷ (1 + interestRate)` 得到折现因子 v
3. `×` — 计算 `sumAssured × qx`

再拖入 1 个 **Operator** `×` — 计算最终结果

### 步骤 4：添加精度控制

拖入 1 个 **Function** 节点，选择 `round`，可在后端配置精度位数

### 步骤 5：连线

```
常量 1 ────→ [+ 加法] ←──── interestRate
               │
常量 1 ────→ [÷ 除法] ←──── [+ 加法的输出]
               │
sumAssured ──→ [× 乘法1] ←── qx
               │
[× 乘法1] ──→ [× 乘法2] ←── [÷ 除法的输出]
               │
          [× 乘法2] ──→ [round]
```

### 步骤 6：保存和测试

1. 点击「保存」创建 v1 (Draft)
2. 在测试面板输入：
   ```json
   {"sumAssured": "1000000", "qx": "0.001", "interestRate": "0.035"}
   ```
3. 点击「计算」，预期结果约为 `966.18`（18位精度）
4. 确认无误后到版本管理页面「发布」

## 10. 国际化

编辑器支持三种语言切换，在导航栏右上角选择：
- 中文 (zh)
- 日本語 (ja)
- English (en)

所有节点类型名称、按钮文字、状态标签均随语言切换自动更新。
